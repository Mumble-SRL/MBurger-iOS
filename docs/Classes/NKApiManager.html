NKApiManager

@interface NKApiManager : NSObject

This class is responsible to make all the networking requests to the Nooko Api, NKManager is built above this class.

  • -callApiWithApiToken:Locale:ApiName:HTTPMethod:Parameters:HeaderParameters:Success:Failure:

    Calls the api to the Nooko endpoint with the parameters specified. At the end of the network call it will be called the success block or the failure block, if passed.

    Swift

    func callApi(withApiToken apiToken: String, locale: Locale?, apiName: String, httpMethod: String!, parameters: [AnyHashable : Any]?, headerParameters: [AnyHashable : Any]?, success: ((NKResponse) -> Void)?, failure: ((Error) -> Void)? = nil)

    Parameters

    apiToken

    The api token used to call the Nooko Api.
    locale

    The locale passed to the api call.
    apiName

    The name of the api to call.
    httpMethod

    The http method used to call the api, it can be ‘GET’, ‘POST’, ‘PUT’, ‘PATCH’, ‘DELETE’, if its value is not recognized the api will be called using the ‘GET’ method.
    parameters

    The parameters passed with the api call.
    headerParameters

    The parameters passed in the header of the request with the api call.
    success

    A block object to be executed when the task finishes successfully. This block has no return value and takes one argument: the response of the api (a NKResponse).
    failure

    A block object to be executed when the task finishes unsuccessfully, or that finishes successfully, but the server encountered an error. This block has no return value and takes a one argument: the error describing the error that occurred.
